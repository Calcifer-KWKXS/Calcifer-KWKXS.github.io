基本概念

Graphviz 简介

Graphviz（Graph Visualization Software的缩写，官网）是一款由 AT&T Research 和 Lucent Bell 实验室开源的图形可视化工具，用简单的文本语言对图形进行描述，可以很方便的绘制结构化的图形。

Graphviz 的三个主要组成部分：

  1. layout 自动化布局工具：dot，neato等；
  2. script脚本：主要包含 graph，node，edge三类实体，以及attributes属性；
  3. APIs：graphviz提供了 C、Java、Python、php等语言的API，以供在其它语言中调用。

DOT 语言

DOT语言是一种图形描述语言。

DOT 基本语法

一个简单的 DOT 文件包含以下内容:

  1. digraph 或 graph 声明该图为 有向图或无向图
  2. 节点和边的定义，用来描述图中的元素
  3. 一些可选的属性设置，用来调整图形的样式

标识符命名规则

  1. 字母、数字、下划线，不以数字开头；
  2. 一个数； [-]?(.[0-9]⁺ [-]⁺(|09.[0-9]*)? );
  3. 可能包含转义引号(“…”")的双引号字符串；
    注：在DOT中的引号字符串中，双引号是唯一的转义字符。
  4. 一个 HTML 字符串。<…>

标签相关属性

节点和边中都可以设置标签，只是放的位置不同。

  1. label：附加到对象上的文本标签（在节点内部/边线上）
  2. xlabel：外部标签，对于节点：将放置在节点外部但靠近节点的位置
  3. labelloc=”t”：标签位置，t：标签位于顶部 / b：底部 / c：居中
  4. fontcolor：标签字体颜色
  5. fontsize：标签字体大小

节点（Node）常用属性

节点属性用于描述节点的样式，写在A [这里]。以下是一些节点常用的属性：

  1. shape：节点形状（如 ellipse、box、diamond 等）
  2. label：附加到对象上的文本标签
  3. width 和 height：宽、高
  4. margin ：设置画布的 x 和 y 边距（以英寸为单位）
  5. color：节点边框颜色
  6. fillcolor：节点填充颜色

完整属性介绍：官方文档-节点属性

shape：节点形状

There are three main types of shapes : polygon-based, record-based and user-defined.
形状主要有三种类型：基于多边形、基于记录和用户定义。

The record-based shape has largely been superseded and greatly generalized by HTML-like labels. That is, instead of using shape=record, one might consider using shape=none, margin=0 and an HTML-like label.
基于记录的形状在很大程度上已被类似 HTML 的标签所取代并大大推广。可以考虑使用 shape=record shape=none 和 margin=0 类似 HTML 的标签取代基于记录的形状。

边（Edge）常用属性

边属性用于描述描述边的样式，写在A -> B [这里]。以下是一些边常用的属性：

  1. color：边的颜色
  2. label：附加到对象上的文本标签
  3. fontcolor：标签字体颜色
  4. fontsize：标签字体大小
  5. style：边样式（如 dashed、dotted 等）
  6. arrowhead：箭头样式（仅对有向图有效）

子图

  1. 子图的三个作用：
    1. 表示图形结构，指定有关图形组件的语义信息，表明某些节点和边应该组合在一起。
    2. 为设置属性提供上下文，例如，子图可以指定其中定义的所有节点的默认颜色为蓝色。
    3. 子图的第三种作用直接涉及某些布局引擎如何对图形进行布局。如果子图名称以cluster_ 开头，则Graphviz将该子图标记为特殊集群子图。如果支持，布局引擎将执行布局，使得属于集群的节点一起绘制，整个集群的绘制包含在一个边界矩形内。请注意，好坏参半的是，集群子图不是DOT语言的一部分，而是由某些布局引擎遵循的一种语法约定。
